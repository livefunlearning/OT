网不够好？试试这几种方法。

2020-03-04

情况一
学生：老师，我的微信说你的网不稳定。
老师：对不起，对不起。

情况二
学生：老师，听不清你说什么，断断续续的。
老师：是我这儿的问题吗？但是昨天我跟另一个学生用Skype都很好。

网络好坏直接影响视频教学效果。虽然LFL网教所用视频工具对网速的要求不高，大部分OT师生所在国的网速也不错，但决定网络质量的不仅仅是网速，再加上如今同一wifi只连一台设备的情况几乎不可见。因此，OT教师清晰全面地了解所处网络环境、提前做好预备方案、提醒学生做必要准备，不仅能保证教学的顺利进行，还能在出现网络问题时迅速有效地做调整。因影响网络质量的因素较为复杂，本文仅说明几种常见情况及解决方式。

视频网教的最低要求

为确保一对一视频教学时有清晰的声画，可流畅地使用屏幕共享、白板，快速传输一两张高清图片，师生双方都需达到如下要求。

上传(Upload) 下载(Download) 延迟(Ping)
网速最低要求Minimum Speed Required 5 Mbps 8 Mbps (即1 MB/s) 0 - 60m

注意：

  • 测速工具可能使用不同单位。小写b(bit)，Mbps，指运营商通常所说的XX兆宽带；大写B(Byte)，MB/s，指下载软件通常说的每秒下载XX兆文件。
  • Mbps(宽带大小)÷8=MB/s(下载速度)

提升或保证网络质量的方法

为达到「网速最低要求」，上课前：

  1. 保证视频设备的电量在70%以上，手机信号满格。

  2. 关闭不需使用网页、后台运行APP、杀毒软件，确保视频软件自身有足够的RAM(运行内存)。

  3. 断开暂时不用设备的网络连接，如智能手表、Switch等。

  4. 用不同方式(插网线、WiFi、手机移动数据)联网并测速，选择上传速度最高的联网方式。

测速结果低于「网速最低要求」的解决方法：

1. 对于网线连接、手机移动数据

考虑是否应升级套餐。国内的家庭宽带光纤、手机移动数据的网速均居世界前列。

2. 对于WiFi连接

  • 尝试插网线。有线网的信号干扰少，视频声画的卡顿比无线WiFi少，更稳定。

  • 重启路由器；换一个房间或靠近路由器。

  • 换路由器。在不更换路由器的情况下，普通单频无线路由器，在信道(channel)1、6、11之间切换(方法)；双频无线路由器，在频段2.4GHz和5GHz之间选择不同的wifi信号连接。更改后，重启路由器。

    （一般来说，5 GHz更快，但需靠近路由器；2.4GHz远距离信号更稳，但默认使用这个频段的设备多，易受干扰出现延时或掉线。）

  • 若在学校、咖啡等公共场所使用WiFi，应避开用网高峰期。如无法避开，可尝试手机流量(mobile data)。

  • 如以上方法仍无法满足「网速最低要求」，做好课上改用语音通话的准备。

  • 以上方法的英文视频版(可发给学生或科学上网观看)：https://www.wsj.com/video/series/joanna-stern-personal-technology/speed-up-your-home-wi-fi/9B472EB7-BC07-4C90-AC57-4C4AF0F53C7F

网络测试工具

  1. 首选：Speedtest by Ookla

    • 优点：师生均可使用
    • 电脑端：浏览器直接测试 下载测试
    • 手机端：浏览器直接测试 下载App(需有非国区苹果账号或能科学上网的安卓手机)

  2. 其次：网速测试大师(英文名Internet Speed Test-speedtest)

    • 优点：教师获取手机app方便
    • 电脑端：无
    • 手机端：苹果、安卓

解读测试数据

1. 是否够快？看上传数值、下载数值、延迟(Ping)。

  • 上传(Update load)：>5 Mbps

  • 下载(Download)：8 Mbps (即1 MB/s)

  • 延迟(Ping)：0 - 60ms


速度不够的表现

  • 一般来说，下载速度应比上传速度快。如两项数字几乎持平，或下载低于上传，则说明此时用网人数较多，应优先保证稳定性。如上图为UNM校园网的网速测试结果。
  • 一方下载速度低于「网速最低要求」：在接收对方发来的高清图片时，加载需更长时间。
  • 一方下载速度高于「网速最低要求」，但上传速度不行：对方可能出现画面卡顿或声音断断续续的情况。


解决方法

  • 尝试上文的「提升或保证网络质量的方法」


2. 是否够稳？看抖动(Jitter)、丢包率(Packet loss)

  • 抖动(Jitter)：0-30ms。

  • 丢包率(Packet loss)：0-5%。。


稳度不够的表现

  • 抖动小，网速平稳；抖动大，网速时好时坏。
  • 丢包率大，路由器被大量占用，有明显的卡顿。


解决方法

  • 插网线

  • 重启路由器

  • 网络丢包的四大原因和修复方法

丢包率测试

网络测试工具不一定会显示丢包率，在此介绍一种简单的测试方式。

  1. 打开电脑
    Windows：开始→搜索→输入 cmd→回车
    Mac：Command+空格→输入 Terminal→回车

  2. 此时Terminal被打开。接着，输入下列文字，再点击回车。

ping -n 50 8.8.8.8

  1. 一分钟后，出现如下文字。
  • Windows

Ping statistics for 8.8.8.8:
Packets: Sent = 50, Received = 50, Lost = 0 (0% loss)

  • Mac

50 packets transmitted, 50 packets received, 0.0% packet loss
round-trip min/avg/max/stddev = 58.761/59.494/60.527/0.413 ms

此时可见丢包率(Packet loss)为0.0% 。
